# TASK 文档模板

```markdown
# TASK: [任务名]

> 注意：TASK 不得记录密钥/Token/个人数据；必要时用脱敏摘要或引用文件路径代替粘贴原文。

## 任务元信息（必选）
- **创建时间**：[YYYY-MM-DD HH:mm]
- **状态**：[进行中/已完成/已暂停]
- **复杂度**：[简单/中等/复杂]
- **预计步骤**：[X个主要步骤]

## 任务目标（可选，建议包含）
[清晰描述任务要达成什么结果，可以包含验收标准]

## 任务背景（可选）
[说明为什么要做这个任务，上下文是什么，解决什么问题]

## 任务设计（可选）
[技术方案、架构设计、关键决策、风险点等]

## 残余风险/待验证项（若存在必填）
- [ ] [当前无法覆盖/无法验证/仍有争议的点]
  - 影响：[可能影响哪些功能/数据/用户]
  - 建议验证：[下一步怎么验证、需要什么条件]

## 实施过程（长任务必选，简单任务可省略）
### 进度追踪
- [x] 步骤1：[描述] - 完成于 [HH:mm]
- [ ] 步骤2：[描述] - 当前进行中
- [ ] 步骤3：[描述] - 待开始

### 下一步行动
- [ ] [具体的下一步任务]
- [ ] [依赖条件或前置要求]

### 并行协作记录（仅当启用子代理/多角色复核时必选）
| 时间 | agent_id/角色 | 分工 | 关键结论 | 证据（文件/命令输出/官方文档） | 采纳 | 备注 |
| ---- | ------------- | ---- | -------- | ----------------------------- | ---- | ---- |
| [HH:mm] | [id] | [做什么] | [结论] | [例如：src/a.ts#L10 / `rg ...` 输出摘要] | [是/否] | |

### 自检-纠偏记录（触发时必选：不确定性/关键决策/失败/范围变化）
#### [HH:mm] 触发：[原因]
- **不确定点**：[为什么不确定/冲突点是什么]
- **纠偏动作**：[补充阅读/补测/拆分范围/回滚等]
- **结论**：[最终选择与理由]
- **后续验证**：[需要验证什么、怎么验证]

### 关键决策
#### [HH:mm] 决策：[标题]
- **背景**：为什么需要做这个决策
- **方案**：选择了什么方案
- **理由**：为什么选择这个方案
- **影响**：会影响哪些部分

### 问题与解决
#### [HH:mm] 问题：[标题]
- **问题描述**：具体遇到了什么问题
- **尝试方案**：尝试过哪些解决方案
- **最终方案**：最终如何解决
- **经验教训**：下次如何避免

## 最终交付（完成后填写）
- [ ] 功能代码
- [ ] 测试或验证
- [ ] 文档更新
- [ ] [其他交付物]

### 验证结果（可选）
[测试结果、性能数据等]
```
