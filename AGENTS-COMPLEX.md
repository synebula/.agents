# 复杂任务执行规范 (AGENTS-COMPLEX)

## 0. 核心原则
- **数据驱动**：先读相关源码/SPEC，严禁盲目猜想。
- **方案优先**：先定方案（Plan），明确风险与验证手段，再动手。
- **任务留痕**：通过 `TASK-*.md` 同步进度与关键决策，确保中断后可恢复。

## 1. 标准工作流
### 1.1 调研 (Research)
- 读取根目录及子目录相关的 `CONTEXT.md` 和 `SPEC.md`。
- 阅读调用方/实现方代码及 ≥2 个相似用例。
- 确认外部依赖 (Library/MCP) 的版本与正确用法。
- **产出**：影响范围清单 + 可复用逻辑/组件。

### 1.2 规划 (Plan)
- 优先定义核心数据结构与公共接口。
- 评估风险：接口兼容性、数据迁移、安全边界、性能影响。
- 制定验证计划：具体的单测、集成测试或手工验证路径。
- **产出**：分步实施计划（含验证点），计划格式遵循 `1. [步骤] → 验证：[检查项]`。

### 1.3 实现 (Implement)
- 每一步完成必须执行预定验证。
- 行为约定、接口契约、运行/验证方式发生变化后，更新最近的 `CONTEXT.md`；跨目录改动更新共同父级，架构调整同步刷新根 `CONTEXT.md`。
- 任务结束需合规清理冗余代码、未使用的导入及配置。

## 2. 过程追踪 (TASK 文档)
**启用条件** (满足其一即启用)：
- 涉及 ≥3 个模块或 ≥3 个实施阶段。
- 涉及高危动作：数据/权限变更、核心 API 签名称重构。
- 启用子代理并行复核时。

**记录要素**：
- 实时进度（已办/当前/待办）。
- 关键方案变更原因及最终决策。
- 问题/失败尝试与最终解决方式。
- 下一步动作、依赖条件与回滚点。
- 自检纠偏结论：范围变化、假设修正、未证实点。
- 启用子代理时，补充分工、证据来源与采纳理由。
- 残余风险与后续验证建议。

## 3. 并行协作 (Sub-Agents)
- **定位**：子代理仅用于只读工作（检索、复核、方案推演）。
- **写操作**：严禁子代理执行任何写操作（文件修改/删除等），统一由主代理负责。
- **双源复核**：高影响结论需 ≥2 个子代理独立推导一致，或证据链（代码 vs 文档 vs 测试）闭环。
- **汇总**：主代理需在 TASK 中汇总子代理关键结论、证据来源与采纳理由。

## 4. 记忆维护 (CONTEXT.md)
- **更新硬规则**：仅记录“现状”（怎么做、在哪改、怎么跑），严禁记录过程日志或变更历史（过程写 TASK，历史写 CHANGELOG）。
- **层级管理**：父目录 CONTEXT 侧重索引导航，子目录 CONTEXT 侧重实现约定。
- **刷新时机**：除架构变更外，只要当前目录的行为约定、接口、入口、运行/验证命令发生变化，就应就近同步刷新，避免快照过期。
